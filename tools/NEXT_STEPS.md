## The core unfinished fix: ID3-on-FLAC defect (3 sites, all approved by user)

**Root cause:** the production tag *reader* everywhere (`taghag_import/flac.py::extract_flac_tags`,
used by `cli.py`, `audio_audit.py`, `stage.py`) reads **Vorbis comments** via `mutagen.flac.FLAC`.
But several *writers* use `mutagen.id3.ID3` (ID3v2 frames) on `.flac` files instead — writes that
are invisible to every real reader. Verified empirically: write ISRC via `apply_flac_tag_updates`,
read back via `flac.py::extract_flac_tags` → `None`.

User-approved fix: switch all three writers below to Vorbis comments via `mutagen.flac.FLAC`,
matching `flac.py`'s field-name convention. Do NOT consolidate the two `sync_vibes` duplicates
into a shared helper — fix each in place (explicit advisor guidance: that's unrequested
refactoring; three similar lines beat a premature abstraction).
