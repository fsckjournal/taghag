  What I built — tools/taghag_import/time_base.py

  reconcile_offset(...) canonicalizes any rendition's cue grid onto the
  master-FLAC clock via canonical_ms = measured_ms − offset_ms, picking the
  strongest applicable method:

  1. identity — source is the canonical FLAC → offset 0, confidence 1.0
  2. cross_correlation (the workhorse) — grid_offset() votes over pairwise diffs
  of the two cue grids: collects every source − canonical within a 60 ms
  window, finds the histogram peak (most diffs within 8 ms), returns mean as
  offset + MAD as residual. Non-corresponding cues scatter and never form a
  peak; only truly-matched cues pile up at the real lag. Pure-Python, no decode.
  3. downbeat_anchor — single explicit anchor beat per grid (sparse fallback)
  4. declared_priming — encoder_delay_samples / sample_rate_hz × 1000

  pcm_cross_correlation() (onset-envelope decode) is librosa-gated — returns
  None when the dep is absent, so the module stays pure-Python (toolchain has no
  numpy/scipy/librosa). Plus db_client.upsert_rendition_time_offsets() for
  persistence.
